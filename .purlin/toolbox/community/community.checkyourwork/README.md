# Check Your Work

After spec or code - check to make sure you created the right solution.

## Usage

```
/pl-toolbox run checkyourwork
```

## What it does

For the work just completed:

1. If specs, make sure they cover all the use cases specified. Make sure the tests meaningfully cover the goals. Make sure there are no ambiguities.
2. If code, make sure the solution is elegant, maintainable, and meets the goals from the specifications.
3. If both, make sure they align and together meet the goals specified.

Presents gaps and suggestions to the user.
