# Implementation Notes: Unified Project Init

*   **Inherits from bootstrap.sh:** The init.sh script absorbs all logic currently in `tools/bootstrap.sh`. Start by reading the existing bootstrap implementation and refactoring it into init.sh with mode detection wrapping the existing logic. The full init path should produce identical artifacts to what bootstrap.sh produces today.
*   **Mode Detection:** The mode check is a single `[ -d "$PROJECT_ROOT/.purlin" ]` test at the top of the script. Full init is the `else` branch; refresh is the `then` branch.
*   **Shim Generation:** The shim is a heredoc written by init.sh. Use `git -C "$SUBMODULE_DIR" remote get-url origin` for the remote URL, `git -C "$SUBMODULE_DIR" rev-parse HEAD` for the SHA, and `git -C "$SUBMODULE_DIR" describe --tags --abbrev=0 HEAD 2>/dev/null || echo "untagged"` for the version. Embed these as comments in the shim header.
*   **CDD Symlinks:** Use `ln -sf` with relative paths. The target should be relative from the project root to the submodule's tools directory (e.g., `purlin/tools/cdd/start.sh`). Test with `readlink` to verify relative.
*   **Command File Timestamp Logic:** Use `-nt` (newer than) test in bash: `if [ "$dest_file" -nt "$src_file" ]; then skip; fi`. This is the same approach used by bootstrap.sh Section 2.18.
*   **`--quiet` Implementation:** Gate all `echo` statements behind a `$QUIET` flag variable. Set `QUIET=true` when `--quiet` is in `$@`. Errors should still go to stderr regardless.
*   **`--regenerate-launchers` Implementation:** Set a `REGEN_LAUNCHERS=true` flag. In refresh mode, the launcher generation block only runs when this flag is set. In full init mode, launchers are always generated regardless.
*   **Bootstrap Deprecation:** Modify `bootstrap.sh` to be ~5 lines: shebang, deprecation message to stderr, resolve init.sh path, exec init.sh with "$@". Preserve the existing file for backward compatibility.
*   **Submodule Root Symlink:** The `init.sh` symlink at the submodule root (`purlin/init.sh -> tools/init.sh`) is a committed symlink in the Purlin repo itself. Create it with `ln -s tools/init.sh init.sh` from the repo root.
*   **Test Script Architecture:** `tools/test_init.sh` should follow the same sandbox pattern as `tools/test_bootstrap.sh` — `setup_sandbox()` function, `trap cleanup EXIT`, overlay of uncommitted changes. Each test is a shell function that prints PASS/FAIL. Run all tests and accumulate results for final summary. Consider reusing sandbox setup across tests to save time (one full init, then refresh tests reuse the same sandbox).
