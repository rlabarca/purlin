## 1. Overview

The PM agent launcher script (`pl-run-pm.sh`) enables launching the PM agent role via the Claude CLI. It reads agent configuration from `config.json`, passes model, effort, and permissions to the CLI, and loads the PM's instruction files. This follows the same launcher pattern as existing `pl-run-architect.sh`, `pl-run-builder.sh`, and `pl-run-qa.sh`.

---

## 2. Requirements

### 2.1 Launcher Script

- The launcher MUST be generated by `tools/init.sh` alongside the other agent launchers.
- The script MUST be placed at `<project_root>/pl-run-pm.sh` (same level as other launchers).
- The script MUST read `agents.pm` configuration from the resolved `config.json`.

### 2.2 Config Reading

- Read the `pm` agent configuration from the `agents` section of `config.json`.
- Default configuration when `agents.pm` is absent:
  ```json
  { "model": "claude-sonnet-4-6", "effort": "medium", "bypass_permissions": true }
  ```
- The PM defaults to Sonnet 4.6 (creative iteration is fast and conversational).

### 2.3 CLI Arguments

- If `model` is set, pass `--model <model>`.
- If `effort` is set, pass `--effort <effort>`.
- If `bypass_permissions` is `true`, pass `--dangerously-skip-permissions`.
- If `bypass_permissions` is `false`, pass `--allowedTools` with PM role restrictions:
  `"Bash(git *)" "Bash(bash *)" "Bash(python3 *)" "Read" "Glob" "Grep"`

### 2.4 Instruction Loading

- The launcher MUST concatenate and pass the following instruction files as the session prompt:
  1. `instructions/HOW_WE_WORK_BASE.md`
  2. `.purlin/HOW_WE_WORK_OVERRIDES.md` (if exists)
  3. `instructions/PM_BASE.md`
  4. `.purlin/PM_OVERRIDES.md` (if exists)

### 2.5 Startup Sequence Config

- The PM uses `startup_sequence: false` and `recommend_next_actions: false` by default.
- The PM is a conversational agent that awaits human direction rather than running automated gap analysis.

## 3. Scenarios

### Automated Scenarios

#### Scenario: PM Launcher Reads Agent Config from Config JSON

    Given config.json contains agents.pm with model "claude-sonnet-4-6"
    And agents.pm has effort "medium" and bypass_permissions true
    When the PM launcher script is executed
    Then it invokes the claude CLI with --model claude-sonnet-4-6 --effort medium
    And it passes --dangerously-skip-permissions

#### Scenario: PM Launcher Falls Back When Config is Missing

    Given config.json does not contain an agents.pm section
    When the PM launcher script is executed
    Then it invokes the claude CLI with model claude-sonnet-4-6 effort medium
    And bypass_permissions defaults to true

#### Scenario: PM Launcher Loads Correct Instruction Files

    Given instructions/PM_BASE.md exists
    And .purlin/PM_OVERRIDES.md exists
    When the PM launcher script is executed
    Then the session prompt includes HOW_WE_WORK_BASE.md content
    And the session prompt includes PM_BASE.md content
    And the session prompt includes PM_OVERRIDES.md content

#### Scenario: Init Script Generates PM Launcher

    Given tools/init.sh is run to bootstrap a new consumer project
    When the init script generates launcher scripts
    Then pl-run-pm.sh is created alongside other agent launchers
    And pl-run-pm.sh has executable permissions
