## 1. Overview

Role-specific launcher configuration for the PM agent (`pl-run-pm.sh`). Inherits all shared launcher mechanics from `agent_launchers_common.md`. The PM is a conversational agent focused on design and spec authoring, defaulting to Sonnet 4.6 for fast, interactive iteration.

---

## 2. Requirements

### 2.1 Tool Restrictions (bypass=false)
*   `--allowedTools "Bash(git *)" "Bash(bash *)" "Bash(python3 *)" "Read" "Write" "Edit" "Glob" "Grep"`

### 2.2 Session Message
*   `"Begin PM session."`

### 2.3 Instruction Files
1.  `instructions/HOW_WE_WORK_BASE.md`
2.  `instructions/PM_BASE.md`
3.  `.purlin/HOW_WE_WORK_OVERRIDES.md` (if exists)
4.  `.purlin/PM_OVERRIDES.md` (if exists)

### 2.4 Default Config
When `agents.pm` is absent from the resolved config:
```json
{ "model": "claude-sonnet-4-6", "effort": "medium", "bypass_permissions": true }
```
The PM defaults to Sonnet 4.6 (creative iteration is fast and conversational).

### 2.5 Startup Sequence Config
*   The PM uses `find_work: true` and `auto_start: false` by default, consistent with other agent roles.
*   The PM is a conversational agent that awaits human direction rather than running automated gap analysis.

### 2.6 Init Script Generation
*   The PM launcher MUST be generated by `tools/init.sh` alongside the other agent launchers (see `project_init.md`).

## 3. Scenarios

### Unit Tests
#### Scenario: PM Launcher Dispatches with Config
    Given the resolved config contains agents.pm with model "claude-sonnet-4-6", effort "medium", bypass_permissions true
    When pl-run-pm.sh is executed
    Then it calls resolve_config.py pm to read agent settings
    And it invokes the claude CLI with --model claude-sonnet-4-6 --effort medium
    And it passes --dangerously-skip-permissions

#### Scenario: PM Launcher Falls Back When Config is Missing
    Given config.json does not contain an agents.pm section
    When pl-run-pm.sh is executed
    Then it invokes the claude CLI with model claude-sonnet-4-6 effort medium
    And bypass_permissions defaults to true

#### Scenario: PM Launcher Assembles Correct Prompt
    Given instructions/PM_BASE.md exists
    And .purlin/PM_OVERRIDES.md exists
    When pl-run-pm.sh is executed
    Then the session prompt includes HOW_WE_WORK_BASE.md content
    And the session prompt includes PM_BASE.md content
    And the session prompt includes PM_OVERRIDES.md content

#### Scenario: Init Script Generates PM Launcher
    Given tools/init.sh is run to bootstrap a new consumer project
    When the init script generates launcher scripts
    Then pl-run-pm.sh is created alongside other agent launchers
    And pl-run-pm.sh has executable permissions
